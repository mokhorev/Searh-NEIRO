# SITE_SNAPSHOT_AUDIT

## 1. Назначение

SITE_SNAPSHOT_AUDIT — будущий внутренний модуль Search-NEIRO для controlled snapshot сайта клиента в рамках WEB_FOOTPRINT_SCAN и BEFORE_EVIDENCE_PACK.

## 2. Цель

Сохранять и анализировать страницы сайта клиента: главная, услуги, контакты, о компании, цены, FAQ, документы, лицензии, отзывы, кейсы, формы, рекламные блоки.

## 3. Что искать

- рекламные ошибки;
- слабые офферы;
- слабые CTA;
- отсутствие клиентских формулировок;
- NAP-несоответствия;
- разные телефоны, адреса, email, реквизиты;
- отсутствие ИНН/ОГРН, документов, лицензий;
- отсутствие или слабость страниц услуг;
- отсутствие FAQ под живые клиентские запросы;
- технические ошибки;
- noindex, robots, sitemap issues;
- canonical/redirect/404 проблемы;
- schema.org / JSON-LD ошибки;
- рекламные пиксели, аналитика и внешние скрипты;
- старые счётчики подрядчиков;
- формы без целей аналитики;
- дубли и мусорные скрипты.

## 4. Будущие выходные файлы

- `site_pages.csv`
- `site_assets.csv`
- `site_scripts.csv`
- `site_forms.csv`
- `site_schema.json`
- `site_findings.md`
- `before/site_snapshot.zip`

## 5. Ограничения безопасности

- только домены клиента;
- whitelist `allowed_domains`;
- `max_depth`;
- `max_pages`;
- `max_bytes`;
- `timeout`;
- без обхода CAPTCHA;
- без прокси-пулов;
- без агрессивного скрейпинга;
- без shell exec с сырым URL;
- не использовать как массовый downloader чужих сайтов;
- статус: roadmap only, not implemented.

## 6. Связь с НейроВижин

Модуль нужен для доказательного вывода: сайт как owned-source не подтверждает релевантность компании живым клиентским запросам, поэтому AI/поиск/карты могут видеть компанию слабее или советовать конкурентов.
